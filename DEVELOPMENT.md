# Development — build hygiene

This is a pure-Rust workspace (axum/tokio server + crates) plus a separate Leptos
dashboard crate. The only thing that gets large is the Rust build cache. Everything
below is **regenerable** — safe to delete any time; the next build recreates it.

## Disposable cache dirs (all gitignored — never commit)

| Path | What it is | Regenerated by |
|------|------------|----------------|
| `target/` | workspace build cache | `cargo build` |
| `dashboard/target/` | dashboard (Leptos) build cache — separate crate, `exclude`d from the workspace | `cargo build` in `dashboard/` / `trunk build` |
| `dashboard/dist/` | Trunk/WASM output | `trunk build` |
| `.playwright-mcp/` | QA browser scratch (console logs, page snapshots) | re-running QA |
| `*.log`, `temp/`, `*.db` | scratch logs / local DBs | — |

## Reclaim disk now

```sh
cargo clean                                          # frees target/
cargo clean --manifest-path dashboard/Cargo.toml     # frees dashboard/target/
```

Check space before/after with `df -h .`. A full clean here frees ~9 GB.

## Why builds stay lean

Both `Cargo.toml` roots set:

```toml
[profile.dev]
debug = "line-tables-only"
```

Full debuginfo is what makes `target/` balloon to multiple GB. `line-tables-only`
keeps panic backtraces pointing at line numbers while dropping the bulk. Bump to
`debug = true` locally only when you actually need a step debugger.

Ship release builds for anything real: `cargo build --release` (and the dashboard
via `trunk build --release`). Keep dev builds lean.

## Don't commit caches

`.gitignore` already covers `target/`, `**/target`, `dist/`, `.playwright-mcp/`,
`*.log`, `.env*`, `*.db`, `.DS_Store`, `temp/`. Verify any path with:

```sh
git check-ignore -v <path>
```
